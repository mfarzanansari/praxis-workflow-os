# Support

## Start here

1. Read [README.md](README.md) and [QUICKSTART.md](QUICKSTART.md).
2. Run `python3 scripts/validate_release.py` on POSIX, or `python scripts/validate_release.py` on Windows.
3. Run the installer with `--dry-run` before applying changes.
4. Search existing GitHub issues before opening a new one.

## Where to ask

- **Reproducible defect:** use the bug report form.
- **New capability or workflow idea:** use the feature request form.
- **New or substantially revised skill:** use the skill proposal form and include baseline evidence.
- **Security issue:** follow [SECURITY.md](SECURITY.md); never post secrets or an exploitable report publicly.
- **Usage discussion:** open a focused issue only when the answer may improve project documentation. Personal consulting and guaranteed response times are not part of the open-source support contract.

## What helps maintainers

Include the Praxis version, host agent/client, operating system, Python version, exact command or prompt, relevant file tree, expected behavior, actual behavior, and a minimal reproduction with private data removed.
