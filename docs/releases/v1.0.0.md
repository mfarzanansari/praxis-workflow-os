# Praxis Workflow OS v1.0.0

## The first stable public workflow contract

Praxis is a profession-agnostic Agent Skills operating system that interviews how a person actually works, compiles a personalized workflow architecture, installs the minimum useful capabilities, and evolves the system from evidence.

## Install

```bash
npx skills add mfarzanansari/praxis-workflow-os --skill '*' -g
```

Claude Code:

```text
/plugin marketplace add mfarzanansari/praxis-workflow-os
/plugin install praxis@praxis-workflow-os
```

## Highlights

- Adaptive one-question-at-a-time workflow interview with durable state.
- Blueprint compiler for constitutions, work streams, skill maps, knowledge models, approval boundaries, rollout plans, and acceptance tests.
- Non-destructive Obsidian and personal-skill setup with dry-run, backup, conflict reporting, and rollback.
- Curated distillation into research, decisions, memory, prompts, and project-state notes.
- Evidence-based retrospective and controlled skill-evolution loop.
- Test-driven skill forge with trigger tests, near-miss negatives, security review, provenance, and semantic versioning.
- Seven profession-diverse example profiles.
- Standard-library-only Python scripts with no network calls or package installation.
- Claude Code marketplace and open Agent Skills CLI installation paths.
- CI across Linux, Windows, and macOS.
- Deterministic release archives, checksums, and GitHub build-provenance attestations.

## Prompt Polish

Prompt Polish remains an independently versioned first-party companion. Praxis owns workflow intent and contracts; Prompt Polish may optionally convert an approved contract into a model-specific prompt. No Prompt Polish dependency is required for the core lifecycle.

## Compatibility

- Agent Skills directory format.
- Python 3.9 or newer for bundled scripts.
- Filesystem access for setup, interview state, and note creation.
- No external Python packages.
- Client-specific tool names or skill-discovery behavior may require adaptation by the host client.

## Validation performed

- Repository validator: 7 skills, 0 structural errors, 0 warnings.
- Release-readiness validator: required files, version alignment, marketplace paths, and clean-tree checks.
- Deterministic unit tests across interview state, profile validation, setup safety, note creation, metrics aggregation, installation, manifests, and archive building.
- Example-profile coverage for software architecture, marketing, visual design, creative direction, content writing, field work, and an Obsidian-agent architecture.
- Model-level behavior evals improved from 45/70 initial with-skill assertions to 70/70 across the latest passing remediation traces, compared with a 19/70 no-skill baseline.
- Three-run trigger testing passed 275/336 runs in a crowded Codex catalog. The result and all limitations are documented in the [evaluation report](../evaluations/v1.0.0/README.md), not generalized to other clients or models.

## Security and trust

- No bundled network calls.
- No package installation.
- Non-overwrite behavior by default.
- Forced replacement requires an explicit backup directory.
- SHA-256 checksums included with release archives.
- GitHub Actions release artifacts receive build-provenance attestations.

## Known limits

- Praxis does not provide a universal opaque long-term memory service. It treats Obsidian as a curated durable knowledge layer.
- Generated personal skills are drafts until tested on real work.
- Calendar, email, project-management, and other integrations depend on the tools available in the host agent.
- Interview quality depends on access to representative artifacts and honest user correction.
- Automatic skill selection is client- and catalog-dependent. The v1.0.0 Codex trigger portfolio measured 81.8% accuracy; open routing work is tracked publicly.
- Claude Code structure validation passed, but an organization policy blocked the model-level Claude behavior run before release.

## Upgrade and rollback

This is the first public release. Keep the downloaded archive or tag as a known-good rollback point. Future incompatible profile or workflow-contract changes will use a major version and include migration instructions.
