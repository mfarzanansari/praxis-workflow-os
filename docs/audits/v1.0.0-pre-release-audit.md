# v1.0.0 pre-release audit

## Decision

**Approve v1.0.0 for public release with disclosed routing limitations and public follow-up issues.** No critical or high-severity repository, script-safety, supply-chain, or release-integrity finding remains open. Automatic skill selection is not perfect and is explicitly outside the zero-known-defect claim: the measured crowded-catalog result is 275/336 (81.8%).

## Scope

The audit covered the complete 100-file release candidate before audit artifacts were added:

- all seven Agent Skills, metadata, references, assets, behavior assertions, and trigger suites;
- nine standard-library Python entry points plus POSIX and PowerShell installers;
- profile schema and all profession-diverse examples;
- GitHub Actions CI and release workflows;
- manifest/archive determinism, install/replace/rollback behavior, and symlink boundaries;
- repository governance, contribution, security, support, research, architecture, quickstart, and release documentation;
- native Codex plugin, Claude plugin, and open Agent Skills CLI discovery paths;
- public GitHub security, Actions, release, and repository settings.

## Material findings closed before release

| Area | Finding | Resolution |
|---|---|---|
| Cross-platform integrity | Windows CRLF conversion made the committed manifest appear stale. | Added `.gitattributes` LF normalization and retained Windows CI. |
| Release integrity | Initial release automation did not fully bind artifacts to a verified annotated tag and immutable draft-first publication. | Split build/publish jobs, denied default permissions, pinned actions by full SHA, validated annotated tag and exact SHA, reconciled exact assets, enabled attestations, and published draft-first. |
| Source boundary | Build/install helpers could accept unsafe symlinks or ambiguous untracked distributable files. | Added symlink rejection, tracked-source checks, staged writes, atomic replacement, backup requirements, and cross-plan rollback. |
| Ignored local residue | After the canonical-root move, manifest/archive traversal included ignored `.tmp` recovery files that are absent from clean clones. | Changed Git-checkout builds to package index-tracked/staged files only, retained fail-closed checks for non-ignored untracked files, and added a privacy regression test. |
| Platform filesystem semantics | macOS exposes `/var` through a root-owned symlink and Windows does not expose POSIX executable bits; both triggered false safety warnings. Windows checkout transformations also made worktree-byte manifest hashes unstable. | Allow only root-level platform aliases while still rejecting deeper/user-controlled symlinks, skip POSIX mode warnings on Windows, and hash canonical Git-index bytes in checkouts. |
| Blueprint authority | Setup approval was documentary rather than byte-bound. | Added content-hashed blueprint approval; profile or document drift invalidates approval. |
| Interview transaction | Event/profile writes could diverge on interruption. | Made state transitions transactional and atomic. |
| Resource control | Metrics aggregation could load unbounded event data. | Switched to streaming aggregation with explicit limits. |
| Skill quality | Initial model run passed only 45/70 with-skill assertions. | Reproduced 25 failures, strengthened general response gates, reinstalled the current plugin cache for each valid iteration, and obtained passing traces for all 70 assertions. |
| Trigger evidence | Trigger files lacked repeated real-client activation evidence. | Ran all 112 queries three times and published the 275/336 result and failure clusters. |
| Native Codex packaging | Repository had no native Codex plugin manifest or per-skill UI metadata. | Added native plugin/marketplace manifests and `agents/openai.yaml` for all seven skills; local install/discovery smoke passed. |
| Documentation hygiene | Install routes, telemetry behavior, approval hashing, and evaluation limits were incomplete. | Documented native clients, Skills CLI telemetry opt-out, approval workflow, release evidence, and known limits. |

## Security and supply chain

- Runtime code uses only the Python standard library and performs no package installation or network calls.
- All stateful helpers default to preview or non-overwrite behavior; destructive replacement requires explicit force plus backup.
- Build inputs are repository-bound, symlinks are rejected, archives are deterministic, and release checksums are generated twice for reproducibility checks.
- Workflow permissions are least-privilege per job; external actions are pinned to full commit SHAs and checkout credentials are not persisted.
- GitHub secret scanning, push protection, Dependabot alerts/security updates, private vulnerability reporting, CodeQL, immutable releases, and GitHub-owned action restrictions are enabled.
- The release workflow produces GitHub artifact attestations and reconciles the final immutable asset set.

The project has no third-party runtime dependency graph, so a synthetic package-manager SBOM would add no runtime components. External development and release tools are versioned in this audit and GitHub workflows; release archives contain source, documentation, examples, and standard-library scripts.

## Model evidence and honest limits

See the complete [v1.0.0 evaluation report](../evaluations/v1.0.0/README.md) and [machine-readable summary](../evaluations/v1.0.0/results.json).

- No-skill behavior baseline: 19/70 (27.1%).
- Initial with-skill behavior: 45/70 (64.3%).
- Latest focused passing remediation traces: 70/70 (100%).
- Real-client trigger selection: 275/336 (81.8%).
- Claude behavior run: blocked by organization policy; no result claimed.
- Client catalog pressure: Codex shortened descriptions to a 2% skill-context budget and emitted non-fatal client warnings.

These results authorize a transparent v1 release of the workflow contracts and safe tooling. They do not authorize claims of universal routing accuracy, universal model compatibility, or validation on every client configuration.

## Release controls still required

1. Regenerate and verify the source manifest and deterministic archives.
2. Rerun all deterministic local gates on the final tree.
3. Commit and push the complete release candidate from the canonical root.
4. Require green cross-platform CI and CodeQL at the exact release SHA.
5. Apply branch and `v*` tag rulesets without bypassing release verification.
6. Push the annotated `v1.0.0` tag and let the immutable release workflow publish exact assets and attestations.
7. Download and independently verify checksums, archives, install smokes, repository parity, rulesets, and release immutability.
8. Open public issues for measured routing limitations and the missing Claude behavior run.
