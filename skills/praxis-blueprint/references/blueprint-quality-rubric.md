# Blueprint quality rubric

Score each dimension 0–2.

| Dimension | 0 | 1 | 2 |
|---|---|---|---|
| Grounding | generic assumptions | some profile use | claims trace to profile/evidence |
| Work model | title/persona | partial tasks | complete primitive chains |
| Scope | bloated or trivial | usable but broad | smallest high-value rollout |
| Skill boundaries | overlapping | mostly distinct | narrow triggers and handoffs |
| Knowledge model | transcript dump | note taxonomy only | write thresholds, provenance, retrieval, expiry |
| Risk | absent | generic approvals | action-specific gates and rollback |
| Human factors | ignored | mentioned | encoded in workflow mechanics |
| Verification | vague quality | partial checks | observable gates and acceptance tests |
| Maintenance | no governance | review cadence | evidence, versioning, rollback, deprecation |
| Adoption | big-bang | phased | first-week proof and expansion criteria |

A blueprint should score at least 16/20 and have no zero in Risk, Verification, or Grounding before setup.
