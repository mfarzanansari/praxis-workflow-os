# Generated skill contract

Every generated personal skill must include:

- valid `name` and trigger-focused `description`;
- profile ID and schema version in metadata;
- clear input and output contract;
- observable quality gates;
- human approval boundary;
- durable residue guidance;
- failure/escalation behavior;
- no unsupported claim that setup has been tested;
- no direct credential or secret content.

Generated skills are personal source artifacts. Keep them in version control or another backed-up location. A client-specific install directory is a deployment target, not the only source of truth.
