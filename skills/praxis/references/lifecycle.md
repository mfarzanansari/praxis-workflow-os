# Praxis lifecycle artifact contracts

## Interview → Blueprint

Required:

- `praxis/profile.json`
- confirmed outcomes and work streams
- known tools and sources of truth
- quality and risk boundaries
- knowledge preferences
- unresolved questions with consequences

## Blueprint → Setup

Required approved artifacts:

- workflow constitution
- work-stream map
- skill map
- knowledge architecture
- automation boundaries
- rollout plan
- acceptance tests

## Setup → Operation

Required:

- install report
- created/skipped/conflicted files
- generated skill inventory
- first work stream selected
- rollback/backup location when files were replaced

## Operation → Distillation

Input:

- completed deliverable or meaningful checkpoint
- sources and decisions used
- corrections and verification evidence
- unresolved risks

Output:

- zero or more durable notes; "nothing worth preserving" is valid

## Operation → Retrospective

Input:

- event log, outputs, corrections, invocation history, or representative examples

Output:

- evidence-backed keep/change/remove decisions
- prioritized experiments
- rollback criteria

## Retrospective → Skill Forge

A skill change may proceed when the review identifies a reproducible recurring failure, missed trigger, duplicated deterministic work, or stale domain reference.
